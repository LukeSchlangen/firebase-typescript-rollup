Firebase & Rollup

I've worked at Firebase since April 2014. During that time I've been fortunate to see this small service that was "just used for toy apps" turn into something used by Fortune 500 companies around the world.

Firebase gives developers the ability to develop without a server. You can synchronize data in realtime with the Firebase Database, authenticate users to social media sites with Firebase Authentication, upload and serve binary files with Firebase Storage, host static content with Firebase Hosting, and most recently, send web push notifications with Firebase Cloud Messaging.

However, these features do not come without a cost. Each feature is a complex snowflake that comes with a cost in kilobytes.

The Firebase JavaScript SDK is a total of 92kb. This is a lot of JavaScript to include all at once. Fortunately, the library is designed for modular importing.
